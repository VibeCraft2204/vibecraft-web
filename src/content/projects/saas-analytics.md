## 프로젝트 개요

데이터브릿지는 여러 SaaS 제품의 사용자 행동 이벤트를 수집·분석하여 제품 팀에 인사이트를 제공하는 분석 도구를 자체 개발하고자 했습니다. 기존에 사용하던 서드파티 분석 도구는 비용이 높고 커스터마이징이 어렵다는 한계가 있었으며, 일 2억 건에 달하는 이벤트 처리를 감당하기 위해 처음부터 대용량을 고려한 아키텍처 설계가 필요했습니다. VibeCraft는 데이터 수집, 처리, 저장, 시각화 전 스택을 설계하고 구현했습니다.

## 도전 과제

일 2억 건 규모의 이벤트를 유실 없이 수집하면서 실시간에 가까운 집계를 지원하려면, 기존 RDBMS로는 불가능한 고성능 데이터 인프라가 요구되었습니다. 집계 쿼리가 복잡할수록 대시보드 로딩 지연이 발생하는 문제를 3초 이내 응답이라는 SLA 기준으로 해결해야 했습니다. 이벤트 스키마가 제품별로 달라 유연한 스키마리스 설계와 쿼리 성능 사이의 균형을 잡는 아키텍처 결정이 필요했습니다.

## 접근 방식

Apache Kafka를 이벤트 스트리밍 레이어로 채택하여 수집 속도와 처리 속도의 차이를 버퍼링하고, 파티션 설계로 수평 확장 가능한 구조를 구성했습니다. ClickHouse의 컬럼형 저장 방식과 벡터화된 집계 연산 능력을 활용하여 복잡한 퍼널 분석과 코호트 쿼리도 수초 내에 처리할 수 있도록 했습니다. Grafana 대시보드를 ClickHouse 데이터소스와 연결하고, 자주 사용하는 지표는 Materialized View로 사전 집계하여 응답 시간을 추가로 단축했습니다.

## 주요 성과

일 2억 건의 이벤트를 평균 지연 5초 이내로 처리하는 파이프라인을 구축하고, 복잡한 집계 쿼리도 3초 이내 응답하는 대시보드를 달성했습니다. 기존 서드파티 도구 대비 월 인프라 비용이 65% 절감되었으며, 커스텀 지표와 퍼널을 자유롭게 정의할 수 있어 제품팀의 데이터 활용도가 크게 높아졌습니다. 시스템 도입 후 데이터 기반 제품 의사결정 사이클이 주 단위에서 일 단위로 단축되었습니다.
