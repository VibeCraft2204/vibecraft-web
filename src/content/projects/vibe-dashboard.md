## 프로젝트 개요

VibeCraft 내부에서 사용하는 분석 대시보드는 jQuery와 Bootstrap 기반의 레거시 코드로 구성되어 있었으며, 기능 추가마다 의존성이 얽혀 유지보수에 상당한 비용이 발생하고 있었습니다. 특히 초기 로딩 시간이 8초를 넘어가는 성능 문제로 실무 활용도가 낮았습니다. 이 프로젝트는 현대적인 프론트엔드 스택으로 전면 재설계하여 개발 생산성과 사용자 경험을 동시에 개선하는 것을 목표로 했습니다.

## 도전 과제

기존 대시보드의 기능 스펙이 문서화되어 있지 않아, 레거시 코드를 역분석하며 요구사항을 파악해야 했습니다. 또한 리팩토링 기간 동안에도 팀원들이 실무에서 기존 대시보드를 계속 사용해야 했기 때문에, 신규 버전을 병행 개발하고 점진적으로 전환하는 전략이 필요했습니다. TypeScript 도입으로 인한 초기 타입 정의 작업도 상당한 공수를 요구했습니다.

## 접근 방식

Vite를 빌드 툴로 채택하여 개발 서버 시작 시간을 10배 이상 단축하고, React Query로 서버 상태 관리를 분리하여 불필요한 리렌더링을 최소화했습니다. TailwindCSS로 디자인 시스템을 정비하고, 컴포넌트 단위 코드 스플리팅과 레이지 로딩을 적극 적용했습니다. 전체 컴포넌트에 TypeScript 타입을 정의하여 향후 기능 확장 시 안전한 개발 환경을 확보했습니다.

## 주요 성과

번들 사이즈가 기존 대비 45% 감소했고, 초기 로딩 속도는 8.2초에서 2.7초로 3배 이상 개선되었습니다. 팀 내 기능 개발 속도가 빨라지고 버그 발생 빈도도 크게 줄었으며, 코드 리뷰 시간도 단축되는 부수 효과가 나타났습니다. 이 리팩토링 경험은 이후 클라이언트 프로젝트의 프론트엔드 아키텍처 표준으로 채택되었습니다.
